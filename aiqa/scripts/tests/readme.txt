Для дашборда три реалистичных варианта — от простого к сложному:

1. Azure DevOps Analytics (бесплатно, без кода)
Встроенные Power BI виджеты прямо в ADO. Подключаешь через Boards → Analytics → создаёшь запрос. Ограниченно гибко, но без разработки.

2. Power BI + ADO connector (бесплатно/дёшево)
Microsoft официально поддерживает коннектор ADO → Power BI. Пишешь WIQL — получаешь таблицу — строишь дашборд. Обновляется по расписанию.

3. Простая HTML-страница из скрипта (минимум кода)
Скрипт генерирует статичный metrics.html вместо print в терминал. Открываешь в браузере — видишь цифры. Без сервера.

Что сейчас разумно — зависит от одного вопроса: кто смотрит дашборд и как часто?

Раз в месяц на митинге → вариант 3 (HTML из скрипта)
Постоянный мониторинг для CTO → вариант 2 (Power BI)
Хочешь быстро без лишних инструментов → вариант 1 (ADO Analytics)
