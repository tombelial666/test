# task-summary — 228818

## Суть простым языком

JotForm стал требовать API key не только для REST-ручек, но и для получения **PDF-версии submission**. Без ключа PDF-скачивание может возвращать ошибку (401/403) или невалидный контент.

## Бизнес-намерение

Восстановить/обеспечить возможность скачать **полную заявку в PDF** для account request, где источник формы — JotForm.

## Техническое намерение

Добавить в AMS серверный путь скачивания PDF, который:

- берёт `FormDataReference` (submission id) из `AccountRequest`;
- делает `GET` к JotForm `server.php?action=getSubmissionPDF&sid=...&apiKey=...` с ключом из конфигурации;
- возвращает PDF как байты (`application/pdf`) клиенту.

## Что считается “готово”

- PDF скачивается успешно для request с валидным submission id.
- При неверном ключе возвращается понятный failure (не “битый PDF”).
- Ссылка на скачивание доступна через контролируемый endpoint AMS и правильно пробрасывается/используется в ETNA_TRADER.
