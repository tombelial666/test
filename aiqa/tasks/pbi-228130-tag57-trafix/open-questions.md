# Открытые вопросы (блокируют полную уверенность)

1. **RQD vs Trafix в одном PR**  
   В восстановленном диффе изменён только **`TrafixOrderConverter`**. Не доказано, что **все** исходящие ордера, помеченные бизнесом как «RQD», проходят через этот класс. **Блокер для:** подписи «готово для RQD» без подтверждения конфигурации маршрутов.

2. **Семантика tag 57 вне `FRAC`**  
   Не исследовалось, не выставляет ли **другой** слой (общий FIX header) tag 57 для notional — в текущем коде Trafix **не** вызывает `SetProperty(57, ...)` для notional. Если на стороне библиотеки есть значение по умолчанию, нужен **capture реального сообщения**.

3. **Базовый PR в Azure**  
   Сравнение делалось с **`origin/dev`**. Если PR 15492 влит в другую целевую ветку или содержит дополнительные коммиты не в `origin`, локальный дифф может **не совпадать** на 100% с файлами в PR.

4. ~~**Точное имя фильтра unit-тестов**~~ — снято: см. `OrderConverterTest.cs`, данные подхватываются через `OrderTestData_*.json`.
