## Дистиллированные бизнес-правила (из задачи + кода)

**Из формулировки задачи (evidence в запросе):**

- Для **дробных** ордеров, уходящих в **RQD/Trafix**, тег **57** должен уходить как **`FRAC`**, иначе обработка некорректна.
- Для **notional / cash amount** ордеров:
  - тег **57** должен быть **пустым / не отправляться**;
  - тег **152** должен содержать **денежную сумму** ордера.

**Подтверждение в изменённом коде (`TrafixOrderConverter`):**

- В заголовок FIX (`SetHeaderValues`, группа header / в тестах отражается как вложенность в tag 0) добавляется **57 = `FRAC`** только если `IsFractional(order)` истинно.
- `IsFractional`: **не** считать дробным ордер с флагом `OrderExecInst.CashAmountOrder` (notional); дробность — когда `decimal.Truncate(Quantity) != Quantity`.
- Ветка notional в `SetOrderFields`: при `CashAmountOrder` выставляются `40` (Market), **`152 = order.Price`** (cash), удаляются `44`, `99`, `38` — т.е. сумма в 152 уже была в логике; новый код не добавляет 57 для такого случая через `IsFractional` (cash имеет приоритет в определении «дробности» для tag 57).

## Допущения

- Маршрут **Trafix** в рантайме использует класс `TrafixOrderConverter` (как в JSON тестов `OrderTestData_Trafix.json`).
- Связка **«RQD/Trafix»** в продукте может означать один FIX-поток/конвертер; **в восстановленном диффе изменён только Trafix**, отдельного конвертера «RQD» в этом диффе нет — при необходимости маршрутизацию RQD нужно подтвердить конфигурацией окружения.
