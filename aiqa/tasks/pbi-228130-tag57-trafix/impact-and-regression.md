# Влияние и регрессии

## Вероятно затронутая логика

- **Исходящие New Order Single (и связанные сообщения, где вызывается тот же `SetHeaderValues`)** для маршрута **Trafix**: появление **57 = FRAC** в header при дробном количестве.
- **Расчёт «целой части» для ветки с 20038/38** в `SetOrderFields`: переход с `(decimal)(int)order.Quantity` на `decimal.Truncate(order.Quantity)` для сравнения с `order.Quantity`. Для типичных положительных объёмов поведение совпадает с отсечением дробной части; отличия возможны в **нестандартных** случаях (например, отрицательные или экстремальные значения — если они вообще допустимы в OMS).

## Риск общего кода (shared path)

- Класс **`TrafixOrderConverter`** наследует **`SimpleExecutorConverter`**. Изменения **локализованы** в Trafix: другие конвертеры напрямую не менялись, но **общий шаблон** `ToMessageNew`/`SetOrderFields` тот же — регрессии по **другим** вендорам из этого диффа **не ожидаются**, если только не копировали логику вручную (в диффе не видно).

## Риск по маршруту (route-specific)

- Регрессии возможны **только для потоков**, реально использующих `TrafixOrderConverter`.
- Если **RQD** в проде использует **другой** конвертер, поведение tag 57 **не изменится** этим PR — это **пробел покрытия**, а не ложный негатив.

## Риск регрессии FIX-тегов

| Тег | Риск |
|-----|------|
| **57** | Новое значение `FRAC` в header для дробных share-ордеров Trafix; возможны проблемы у стороны приёма, если формат/место tag 57 не соответствует их спецификации (снижается проверкой на стенде + захватом сообщения). |
| **38 / 20038** | Косвенно: смена способа вычисления `integralQty` — стоит прогнать сценарии с дробным количеством **без** notional. |
| **152** | Для notional логика уже была; регресс — если тесты/ожидания по JSON изменились и что-то ещё читает 152 для не-notional. |

## Что перепроверить вне «ровно нового» поведения

- **Целочисленные ордера Trafix** (tag 57 не должен становиться FRAC).
- **Notional** — отсутствие 57 в заголовке (или пусто), **152 = сумма**.
- **Модификация и отмена** (если тот же header path) — не ломается ли сериализация.
- **Мультилеги** — в диффе нет явных изменений под legs; если Trafix для мультилегов используется, нужен хотя бы смоук.
