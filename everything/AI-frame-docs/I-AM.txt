<goal>
Построить масштабируемый AI QA Framework, который:

1. Анализирует кодовую базу
2. Выявляет изменения (diff / PR / commits)
3. Определяет затронутую логику
4. Генерирует:
   - тест-кейсы
   - автотесты
   - acceptance criteria
   - release notes
5. Оценивает покрытие и риски
6. Интегрируется в CI/CD
</goal>

---

<framework_requirements>

1. Архитектура
   - модульная (agents / skills)
   - расширяемая
   - независимость компонентов

2. Основные модули (agents):

   A. Code Analysis Agent
   - парсит репозиторий
   - строит dependency graph
   - понимает доменную логику

   B. Change Detection Agent
   - анализирует PR / diff
   - определяет impacted areas

   C. Requirement Extraction Agent
   - извлекает требования из:
     • PR описаний
     • тикетов
     • кода
     • комментариев

   D. Test Generation Agent
   - генерирует:
     • unit tests
     • integration tests
     • e2e тесты
     • edge cases

   E. Risk Analysis Agent
   - определяет:
     • критичные зоны
     • legacy зоны
     • нестабильные участки

   F. Coverage Evaluation Agent
   - сопоставляет:
     requirements → tests
   - находит пробелы

   G. Documentation Agent
   - генерирует:
     • release notes
     • readme
     • acceptance criteria

</framework_requirements>

---

<engineering_principles>

Используй best practices:

- Test Pyramid
- Shift-left testing
- Risk-based testing
- FIRST (для автотестов)
- AAA (Arrange / Act / Assert)
- SOLID (для архитектуры)
- DRY / KISS

CI/CD:
- интеграция в pipeline
- автоматический запуск анализа на PR

</engineering_principles>
